# 知识库生成器 - 全局规则

> **范围**: 所有阶段 | **优先级**: 最高 | **版本**: 1.1

---

## 🎯 语言标准

- **严格中文**：思考并使用简体中文响应；避免“翻译腔”。
- **技术术语**：保留英文技术名称（React, JSX, TypeScript, MobX 等）。

---

## 🏛️ 与 TypeScript 定义的关系 (共生)

`AGENTS.md` 不是 TS 定义的副本，它是其**语义补充**。

- **禁止复制定义**：不要列出 Interface 或 Type 的所有属性。
- **引用式链接**：使用“定义见 `path/to/file.ts` 中的 `SymbolName`”来建立链接。
- **专注于非类型知识**：解释属性之间的**约束关系**、**默认值的业务含义**以及**属性变更产生的副作用**。

---

## 💎 知识价值密度 (拒绝废话)

每一份 `AGENTS.md` 必须通过“价值测试”：**资深开发者是否可以在不阅读源码的情况下，通过此文档理解该模块的“灵魂”？**

| 禁止 (肤浅) | 要求 (高价值) |
| :--- | :--- |
| **同义反复**: "Button 组件渲染一个按钮。" | **设计意图**: "封装 AntD Button 以注入全局 Onetrack 点击埋点逻辑。" |
| **重复类型**: "包含属性: id, name, type。" | **数据转换**: "将后端的 UnionQuery 协议转换为 UI 依赖计算所需的扁平化拓扑 Map。" |
| **模糊逻辑**: "通过 API 获取数据。" | **执行机制**: "使用 `IntersectionObserver` 延迟 API 调用，直到面板进入视口 400ms 后触发。" |
| **空白章节**: "数据结构详见 TS。" | **结构性关联**: "通过 `registerLocalFilterChanger` 维护双重状态（局部 vs 全局），以支持‘点击即搜索’模式。" |

**质量自检金标准**：
- **行数预警**：基础模板已包含大量结构化信息，若产出的 `AGENTS.md` **不足 100 行**，通常意味着遗漏了必要的业务章节或代码分析深度不足。在此情况下，你必须重新检查是否缺失了 `## 渲染逻辑`、`## 业务逻辑` 或复杂的 `## 交互事件` 分析。
- **模板反思**：严禁仅对模板占位符进行词典式替换。

## 🗺️ 分层文档策略 (必要性决策)

AI 必须根据以下等级体系决定是否创建独立的 `AGENTS.md`：

| 等级 | 类别 | 必要性 | 规则 |
| :--- | :--- | :--- | :--- |
| **第一级** | **Module (模块)** | **强制** | 定义业务边界和全局数据流。 |
| **第二级** | **Page (页面)** | **强制** | 解释路由、参数和主要页面入口逻辑。 |
| **第三级** | **Business Component (业务组件)** | **按需** | 仅当逻辑复杂、有副作用或存在深层状态耦合时生成。 |
| **第四级** | **Hook / Util / Generic** | **条件触发** | **默认跳过或合并**。但如果逻辑极其复杂、涉及复杂算法或提供关键基础设施，则生成独立文档。 |

**决策逻辑**：生成优先级：复杂度 > 战略重要性 > 类别等级。如果一个第四级目录“一眼看不透”，则必须生成。

## 🚫 拒绝幻觉规范 (数据锚定)

| 规则 | 说明 |
| :--- | :--- |
| **精确复制** | 标识符（ID、类名、方法名）必须从源代码中**精确复制**。 |
| **严禁猜测** | 未经验证的技术名称必须标记为 `[待验证]`。 |
| **来源归属** | 推测性内容标记为 `[推测]`；基于证据的内容需指出证据来源（如：见 X 文件第 N 行）。 |

## 🚫 路径与目录约束

| 严禁行为 | 正确做法 |
| :--- | :--- |
| ❌ 写入源代码目录 | ✅ 输出到 `.knowledge-base/` 目录 |
| ❌ 修改现有源文件 | ✅ 仅创建/更新 AGENTS.md 文件 |
| ❌ 在指定路径外创建文件 | ✅ 严格遵守 `.knowledge-base/` 结构 |
| ❌ 创建其他文件类型 | ✅ 仅允许创建 `.json` 和 `.md` 文件 |

**指定输出路径：**
- `.knowledge-base/*.json` - 阶段性中间文件。
- `**/AGENTS.md` - 生成的文档文件。

## 🚫 禁止操作

| 行为 | 原因 |
| :--- | :--- |
| ❌ 创建临时脚本 | 请使用现有工具脚本。 |
| ❌ 内联代码块 | AI 应分析代码并转化为文档描述，而非搬运代码。 |
| ❌ 使用 Mermaid 图表 | 使用表格、列表或文字描述替代，确保文本可搜索性。 |
| ❌ 仅使用占位符 | 每一句话都必须基于真实的源码分析。 |

## ✅ 执行标准

- **步骤播报**：在开始每个阶段前输出步骤标题。
- **静默处理**：不要对中间过程进行冗长的描述。
- **Token 优化**：大文件仅读取目标章节。
- **质量门控**：在标记任务完成前，必须通过 `validate_agents.js` 校验。

---

## 🔄 并行处理规则

当多个 AI 实例协作时：
1. **先读清单**：在开始前加载 `05-generation-checklist.json`。
2. **认领任务**：找到第一个 `pending` 任务，更新状态为 `in_progress` 并记录时间戳。
3. **互斥工作**：仅处理自己认领的任务。
4. **标记完成**：完成后更新状态为 `completed`。

- ✅ 校验报告显示 0 个错误。
