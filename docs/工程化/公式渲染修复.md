# 公式渲染修复全历程

> 两次 commit：`8e32e8b` → `993dbfc`

---

## 问题起点

用户发送「推导平方根公式」，AI 回复出现三类异常：

| 现象 | 截图表现 |
|---|---|
| `\frac` → `frac`，`\sqrt` → `sqrt` | 反斜杠丢失，LaTeX 命令显示为裸文本 |
| `\(f(x)\)` 行内公式破坏段落 | `\)` 残留，文字换行到公式下方 |
| 流式时页面卡住 → 最后一次性全显示 | 假死后突然刷新全量内容 |

---

## 第一轮修复（commit `8e32e8b`）

### 方向一：后端 SSE 流处理

**假设**：TCP 分片导致 `data:` 行跨两个 `read()` chunk 被截断，反斜杠丢失。

**修复**：在 `chat.service.ts` 引入 `lineBuffer` 跨 chunk 保留未完成行：

```typescript
lineBuffer += value
const lines = lineBuffer.split('\n')
lineBuffer = lines.pop() ?? ''  // 未完成行留到下次
```

同时增加 token 合批缓冲（50 字符 / 40ms）。

→ **结果：方向错误。** LLM 本身就是把 `\frac` 拆成 `\` 和 `frac` 两个 token 输出，不是跨 chunk 问题。

### 方向二：前端 `\(...\)` 定界符错误转换

**问题**：`replaceDelimiters` 把所有定界符统一转 `$$...$$`，`remark-math` 将其视为块级公式，破坏行内排版。

**修复**：区分行内 / 块级，`\(...\)` → `$...$`（行内），`\[...\]` → `$$...$$`（块级）。

→ **结果：有效。** 行内公式不再破坏排版。

### 方向一补充：`trimUnclosedDelimiters`（首次引入）

**目的**：流式中末尾未闭合的 `$$` 会让 remark-math 吞噬后续正文。

**实现**：流式时调用 `trimUnclosedDelimiters`，**裁剪（删除）** 末尾未闭合定界符。

## 第二轮排查 — 流式假死

修复后用户反馈「仍然卡住 → 最后全部显示」。`trimUnclosedDelimiters` 有两个 Bug：

### Bug A：正则匹配错误，大量内容被误删

```typescript
// 这行正则匹配的是「\ + 任意字符 + 一切到末尾」，而非「\[」
let result = text.replace(/\\[([\s\S]*$/, '').replace(/\\\([\s\S]*$/, '')
```

`\f`（来自 `\frac`）、`\s`（来自 `\sqrt`）都会触发删除，把整段公式后的内容清空 → 页面变空白。

**修复**：删除这行有害正则。`\(` / `\[` 已由 `replaceDelimiters` 转换，无需再处理。

### Bug B：删除策略 → 补闭合策略

**旧策略**：奇数个 `$$` → 找到最后一个 `$$`，删除它及其后全部内容。

流式时公式 `$$\nCaO + H_2O...` 未闭合 → `$$` 后全部内容消失 → 页面停留在公式前位置不动。

**新策略**：奇数个 `$$` → 末尾补 `\n$$` 临时闭合，公式内容保持可见。

## 第三轮 — RAH 聚合节流（参考 mimo-chat）

问题仍未完全解决，用户指出参考 `C:\Users\李晓伟\repos\mimo-chat` 的处理方式。

### mimo-chat 核心方案

```
SSE 层：message token → cachedContent（不立即产出）
         → scheduleRafFlush() → 每帧最多 flush 一次
消费层：Promise notify() 唤醒
React 层：rehypeKatex 始终开启 + blockMode freeze 已完成块
```

关键思想：**RAF 聚合 → 每帧只有一次 React 更新 → 一次 KaTeX 渲染**。

### 我们的改动

**1. `service/chat/index.ts`**

| 维度 | 旧方案 | 新方案（参考 mimo-chat） |
|---|---|---|
| 消费端等待 | `requestAnimationFrame(check)` 轮询 | `await new Promise(resolve => resolveWaiter = resolve)` |
| 唤醒 | — | `enqueue()` 时 `notify()` 唤醒 |
| 控制事件 | 直接入队 | 先 `flushCache()` 再 `enqueue`，保证正文先于控制信号 |
| token 产出 | `scheduleFlush`（50 字符 / 40ms） | `scheduleRafFlush`（每帧最多 1 次，对齐浏览器渲染帧） |

**2. `Markdown/index.tsx`**

| 维度 | 旧方案 | 新方案（参考 mimo-chat） |
|---|---|---|
| 流式时 KaTeX | 禁用（公式显示裸 LaTeX） | 始终开启 |
| 性能保障 | 靠禁用 KaTeX 降低开销 | 靠 RAF 节流 + blockMode 控制频率 |

依赖链：

```
RAFT 聚合 → 每帧 1 次更新 → blockMode tail block 仅尾部增量重解析 → KaTeX O(1)
```
